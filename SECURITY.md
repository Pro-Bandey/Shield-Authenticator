# Security Policy

## Supported Versions

We actively support the latest version of the extension available on the Chrome, Firefox, and Edge extension stores. Users are strongly encouraged to keep their extension up to date to ensure they receive the latest security fixes.

## Reporting a Vulnerability

If you discover a potential security vulnerability, please report it privately via our security advisory form:
[Form](https://github.com/Authenticator-Extension/Authenticator/security/advisories/new)

When submitting a report, include as much detail as possible, such as:

- A clear description of the issue
- Steps to reproduce the vulnerability
- A proof-of-concept (if available)
- Any relevant logs, screenshots, or environment details

We aim to acknowledge reports within five (5) days. Please note that this is a volunteer-run project, so response and resolution times may vary.

## Disclosure Policy

We request that vulnerabilities are not disclosed publicly until they have been reviewed and addressed. We will coordinate with you on responsible disclosure and, if applicable, credit you for your report.

## Scope

This policy applies only to the official extension distributed through supported browser stores. Issues arising from unofficial builds, forks, or third-party modifications are outside the scope of this policy.
